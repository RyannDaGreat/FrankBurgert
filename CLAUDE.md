# CLAUDE.md

## About this project

This is the FSC (Frank S. Consulting) website, hosted on GitHub Pages.

## User context

The primary user (Frank) is not technical — he has no experience with git, and his computer setup can be unreliable. Always:

- **Auto-commit and push** after every change. Never ask "do you want me to commit?" — just do it.
- **Explain nothing about git** unless he specifically asks. Don't mention branches, staging, remotes, etc.
- **Be patient and forgiving** with typos, vague instructions, and unclear requests. Interpret intent generously.
- **Don't ask unnecessary questions.** If something is obvious or can be reasonably inferred, just do it.
- **Keep things simple.** Avoid jargon. If something breaks, fix it quietly and move on.
- **Always push to remote** after committing — don't wait to be asked.
